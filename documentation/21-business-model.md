# 12 — Business Model

## Business model summary

Mody Cloud should begin as a monthly subscription hosting business and expand into a developer ecosystem with higher-value platform, workflow, and service revenue.

The first product is intentionally simple:

- One hosted Bari WordPress site.
- 3GB storage.
- Monthly recurring subscription.

## Revenue streams

### 1. Hosting subscriptions

Core recurring revenue from hosted Bari sites.

Early subscription unit:

- One site.
- One Bari stack.
- 3GB storage.
- Managed baseline.

### 2. Storage upgrades

Customers with media-heavy or growing sites can buy more storage.

Potential storage add-ons:

- Additional 5GB.
- Additional 10GB.
- Larger pooled agency storage.
- Object-storage-backed media package.

### 3. Performance upgrades

Customers can pay for better runtime resources.

Examples:

- More PHP memory.
- More PHP workers.
- Better CPU allocation.
- Enhanced Redis/cache setup.
- CDN package.
- Image optimization package.

### 4. Workflow upgrades

Developer-focused paid features:

- Staging environments.
- Git deployment.
- Preview deployments.
- Migration runner UI.
- Team access.
- Audit logs.
- Advanced environment controls.

### 5. Agent-assisted services

Paid services using Bari's agent-first structure:

- Site setup assistance.
- Documentation generation.
- Plugin scaffolding.
- Migration review.
- Security review.
- Maintenance summaries.
- Release note generation.
- Support ticket triage.

### 6. Marketplace revenue

Longer-term revenue from curated ecosystem assets:

- Premium patterns.
- Site kits.
- Bari-compatible plugins.
- Native block packs.
- Agent workflow packs.
- Agency templates.

## Cost drivers

Major cost drivers:

- Compute resources.
- Database resources.
- Storage for uploads.
- Backup storage.
- Bandwidth.
- Monitoring/logging.
- Support time.
- Billing/payment fees.
- Control panel development.
- Security tooling.

## Unit economics to model

Per site:

- Average storage usage.
- Backup multiplier.
- Average bandwidth.
- Average CPU/memory usage.
- Average database size.
- Support minutes per month.
- Gross margin.

Per account:

- Number of sites.
- Average revenue per account.
- Churn.
- Expansion revenue.
- Support burden.

## MVP assumptions to validate

The MVP should validate:

- Developers understand the value of Bari-powered hosting.
- 3GB is enough for starter sites and landing pages.
- Customers are willing to pay for a modern workflow, not just raw hosting.
- Support load is manageable with a standardized stack.
- Agent-first documentation reduces maintenance time.
- Storage and performance upgrades are natural expansion paths.

## Expansion logic

The business should expand in layers:

1. Host one Bari site reliably.
2. Add more sites/accounts.
3. Add storage and performance upgrades.
4. Add workflow features for developers.
5. Add team and agency features.
6. Add agent-assisted services.
7. Add marketplace/community revenue.

## Risks

### Commodity hosting pressure

Risk: customers compare Mody Cloud to cheap shared hosting.

Response: position around developer workflow, Bari architecture, agent-readiness, and managed modern WordPress delivery.

### Support overload

Risk: WordPress plugin conflicts create too much support burden.

Response: define support boundaries, offer paid debugging, and maintain a curated compatibility stance.

### Infrastructure complexity

Risk: advanced controls create operational risk too early.

Response: start simple, expose controls progressively, validate every setting change, and maintain audit logs.

### Ecosystem quality

Risk: marketplace/community assets become inconsistent.

Response: curate heavily, require documentation, and enforce Bari compatibility standards.

## Strategic moat

Mody Cloud's moat should come from:

- Bari as a repeatable technical foundation.
- Developer trust.
- Agent-first conventions.
- Community knowledge.
- Reusable patterns/plugins.
- Operational experience hosting Bari sites.
- A control panel that understands WordPress and Bari-specific environment settings.

## Business north star metrics

- Monthly recurring revenue.
- Active hosted Bari sites.
- Active developers.
- Gross margin per site.
- Time to provision a site.
- Time to first successful launch.
- Support tickets per site.
- Upgrade rate.
- Churn rate.
- Community contributions.
- Agent-assisted task success rate.
