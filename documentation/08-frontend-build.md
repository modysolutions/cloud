# Frontend Build

Mody Cloud compiles frontend assets using **`@wordpress/scripts`** (wp-scripts), which wraps Webpack 5 with a WordPress-optimised default configuration, following the Barí frontend build conventions. The setup is extended in `config/webpack/webpack.config.js`.

The build pipeline is **centralised at the project root** — all custom styles and scripts for the theme and plugins are authored in `src/` and compiled to a single output directory. No individual theme or plugin maintains its own build pipeline.

---

## Source Directory Structure

All frontend source code lives in the `src/` directory. It is organized into two main subdirectories: `theme` and `plugins`.

```
src/
├── theme/
│   ├── app.js
│   └── app.scss
└── plugins/
    └── my-plugin/
        ├── app.js
        └── app.scss
```

- **`src/theme/`**: Contains the main JavaScript and SCSS entry points for the theme.
- **`src/plugins/`**: Contains subdirectories for each custom plugin that requires its own frontend assets. It is recommended to create a new directory for each plugin to keep the assets isolated and organized.

---

## Entry Points

The `package.json` at the root of the project compiles everything in the `src/` directory. The `config/webpack/webpack.config.js` file is used to manage the source entries and output.

| Entry | Source | Output | Purpose |
|---|---|---|---|
| `theme` | `src/theme/app.js` | `app/dist/theme.js` + `theme.css` | All theme frontend assets |
| `my-plugin` | `src/plugins/my-plugin/app.js` | `app/dist/my-plugin.js` + `my-plugin.css` | Assets for `my-plugin` |

Each entry point imports its corresponding SCSS file (and any other JS modules). The SCSS is extracted into a separate CSS file by the MiniCssExtractPlugin included in wp-scripts defaults.

---

## Output Location

All compiled assets are written to **`app/dist/`**, which is web-accessible at `https://{host}/dist/`.

| File | Purpose |
|---|---|
| `dist/theme.js` | Compiled theme JavaScript |
| `dist/theme.css` | Compiled theme CSS |
| `dist/theme.asset.php` | Version hash + dependency list (used by PHP enqueue) |
| `dist/my-plugin.js` | Compiled JavaScript for `my-plugin` |
| `dist/my-plugin.css` | Compiled CSS for `my-plugin` |
| `dist/my-plugin.asset.php` | Version hash + dependency list for `my-plugin` |

The `.asset.php` file is generated by `@wordpress/dependency-extraction-webpack-plugin` and is used to enqueue assets with automatic cache-busting.

---

## npm Scripts

Defined in `package.json` at the project root:

```bash
# Development (watch mode, source maps, fast rebuilds)
pnpm start

# Production build (minified, optimised, no source maps)
pnpm build
```

Both commands use the custom webpack config via `--config-file=config/webpack/webpack.config.js` and write output to `app/dist/` via `--output-path=app/dist`.

## Webpack Configuration

`config/webpack/webpack.config.js` extends the wp-scripts defaults to handle multiple entry points.

```javascript
const path = require('path');
const glob = require('glob');
const defaults = require('@wordpress/scripts/config/webpack.config.js');

// Function to get entries for plugins
const getPluginEntries = () => {
    const entries = {};
    const pluginDirs = glob.sync('./src/plugins/*/');
    pluginDirs.forEach(dir => {
        const pluginName = path.basename(dir);
        entries[pluginName] = path.resolve(process.cwd(), dir, 'app.js');
    });
    return entries;
};

module.exports = {
    ...defaults,
    entry: {
        theme: path.resolve(process.cwd(), 'src', 'theme', 'app.js'),
        ...getPluginEntries(),
    },
    module: {
        ...defaults.module,
        rules: [
            ...defaults.module.rules,
            // Images → copied to dist/ as static assets
            {
                test: /\.(png|svg|jpg|jpeg|gif)$/i,
                type: 'asset/resource',
            },
            // JS/JSX → Babel with React preset
            {
                test: /\.(js|jsx)$/,
                exclude: /node_modules/,
                use: {
                    loader: 'babel-loader',
                    options: { presets: ['@babel/preset-react'] },
                },
            },
        ],
    },
};
```

This configuration automatically finds all plugins in the `src/plugins` directory and creates a Webpack entry for each one.

## Plugin and Theme Development

When developing a new plugin that requires frontend assets, you should:

1.  Create a new directory for your plugin inside `src/plugins/`.
2.  Add your `app.js` and `app.scss` files to this new directory.
3.  The Webpack build process will automatically pick up the new entry point and create the corresponding asset files in `app/dist/`.
4.  Enqueue the generated assets in your plugin's PHP code, using the `.asset.php` file for dependencies and versioning.

This approach keeps the frontend assets for each plugin isolated while still benefiting from a centralized build process.
